## Context
Tournament-MCP Integration: How should the architect skill integrate with tournament-mcp for high-quality proposal ranking? The tournament-mcp provides Swiss-Lite scheduling and Bradley-Terry ranking but is an external MCP server. Context: Solo developer, architect is only consumer, Skills run in code execution container, MCPs are external servers Claude talks to.

## Decision
Architect skill integration with tournament-mcp via CLI interface provides self-contained skill while leveraging external tournament tool. Follows established pattern from research skill (calls gemini/codex CLIs). Tournament installed globally via uv tool install, skill remains dependency-free. Codex analysis confirmed this approach balances maintainability, performance, coupling quality, reusability, testing ease, and clarity. CLI overhead negligible for 2-3 minute tournaments. Tournament-mcp provides both MCP interface (for non-Python consumers) and CLI interface (for bash scripts). Winner: Option B (CLI approach) from architecture analysis.
