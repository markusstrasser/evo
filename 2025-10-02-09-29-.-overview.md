# Project: evolver

## Quick Start

-   **Install:** `npm install`
-   **Run Dev Server:** `npm run dev` (starts shadow-cljs watch)
-   **Run Tests:** `npm test`

## Project Structure

-   **`dev/`**: REPL helpers, test fixtures, health checks, and local development tooling. Not for production code.
-   **`docs/`**: Project documentation, research specs, architectural decision records (ADRs), and prompt templates.
-   **`scripts/`**: Automation and workflow scripts for research, analysis, code generation, and testing.
-   **`mcp/`**: Model Context Protocol (MCP) server implementations for AI agent integration.
-   **`public/`**: Static assets for the front-end application, including `index.html`.
-   **`.legacy_deprecated/`**: Archived code that is no longer in use but kept for historical context.

## Development Workflow

-   **REPL Setup:** Connect your editor to the nREPL server on port `55449` (defined in `shadow-cljs.edn`). Use `dev/repl/init.clj` for setup helpers.
-   **Testing Strategy:** Run unit tests via `npm test`. Test configuration is in `tests.edn`. Use fixtures from `dev/fixtures.cljc` to generate test data.
-   **Quality Gates:**
    -   **Linting:** Run `npm run lint` to check code with `clj-kondo`.
    -   **Pre-commit:** A hook at `.pre-commit-check.sh` runs linting, compilation, and tests before committing. Install with `scripts/install-hooks.sh`.
-   **Debugging Tools:**
    -   **Health Checks:** Run `npm run agent:health` or use `dev/health.clj` for environment diagnostics.
    -   **Error Reference:** `dev/error-catalog.edn` contains a taxonomy of common errors and their remedies.

## Documentation & Context

-   **`README.md`**: High-level project philosophy and architectural goals.
-   **`CLAUDE.md`**: The primary guide for AI agents, detailing workflows, tooling, and key files.
-   **`STYLE.md`**: Clojure style guide with project-specific patterns and anti-patterns.
-   **`docs/`**: Contains deep dives into the architecture (`MCP.md`, `EVALUATOR.md`, `MLIR. UI SYSTEM as COMPILER`).
-   **Research Materials:** Research workflows, prompts, and results are managed under `docs/research/` and `scripts/`.

## Tooling & Automation

-   **NPM Scripts (`package.json`):**
    -   `npm run dev`: Start the development server.
    -   `npm test`: Run the test suite.
    -   `npm run lint`: Lint the codebase.
    -   `npm run fix:cache`: Clear all build and dependency caches.
-   **Shell Scripts (`scripts/`):**
    -   `architectural-proposals`: Generate architectural proposals from multiple LLMs.
    -   `best-repos-research`: Run parallel research queries against reference repositories.
    -   `generate-overview.sh`: Create AI-generated overviews of the codebase.
-   **Git Hooks:** A `pre-commit` hook is defined in `scripts/hooks/pre-commit`.
-   **MCP Servers:**
    -   Defined in `.mcp.json` and `deps.edn` (aliases `:mcp` and `:mcp-shadow-dual`).
    -   Implementation for dev diagnostics is in `mcp/servers/dev_diagnostics.clj`.

## Common Workflows

-   **Adding Features:** Follow existing patterns, add unit tests, and ensure `npm run check` passes.
-   **Running Research:** Use scripts like `scripts/best-repos-research` and `scripts/architectural-proposals`. See `CLAUDE.md` for detailed workflows.
-   **Debugging Issues:** Use `dev/health.clj` and `dev/bin/health-check.sh` to diagnose environment issues. Refer to `dev/error-catalog.edn` for common failure patterns.

## Key Files Reference

-   **`deps.edn`**: Clojure dependencies and aliases for running different project tasks (nREPL, MCP servers, tests).
-   **`package.json`**: Node.js dependencies and scripts for the development workflow (linting, testing, building).
-   **`shadow-cljs.edn`**: Configuration for the ClojureScript compiler, including source paths, dependencies, and build targets.
-   **`.mcp.json`**: Defines local and remote MCP servers available for AI agent integration.
-   **`CLAUDE.md`**: Central reference for AI agents on how to interact with the project.
