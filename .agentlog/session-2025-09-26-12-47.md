## Failure Modes & Gotchas

### ClojureScript REPL Development
- **Shadow-cljs Process Conflicts**: Running `npm dev` already starts shadow-cljs; manual `npx shadow-cljs` commands create conflicts
- **Test Execution**: `npm test` often hangs; CLJS REPL with `(cljs.test/run-tests)` more reliable
- **Macro Resolution**: Cross-namespace macro imports need explicit `:refer-macros` declarations
- **Store State Pollution**: REPL state persists between test runs; use fresh test stores for isolation

### Testing & Validation
- **Test Isolation**: Integration tests can affect each other due to shared store state
- **Agent Tool Loading**: Requires browser context (`detect-environment :store-accessible?`) for full functionality
- **Test Runner Fragility**: Node-based test execution less reliable than REPL-based testing

### Development Workflow Issues
- **File Change Detection**: Sometimes requires explicit `:reload` even with hot reloading
- **Error Context**: ClojureScript error messages often lack sufficient context for debugging
- **Meta-data Access**: Attempted to use `(meta pos)` pattern failed; had to pass `db` explicitly to intent builders

## Key Learning: Testing Strategy
- **REPL-First**: Use ClojureScript REPL for development and testing over npm scripts
- **Agent Integration**: Leverage agent tools for store validation and debugging
- **Fresh State**: Always use `helpers/create-test-store` for test isolation
- **Incremental Validation**: Test each intent builder individually before integration

## Future Agent Guidance
1. **Always check for running shadow-cljs processes** before starting development
2. **Use CLJS REPL for testing** instead of npm test when possible  
3. **Test intent builders separately** from command dispatch for easier debugging
4. **Reload namespaces explicitly** when making significant structural changes
5. **Validate with agent tools** after major refactors to catch integration issues
