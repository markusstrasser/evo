# Session Summary: Gotchas & Future Agent Tooling Needs

## Major Failure Modes Encountered

### 1. **Cross-Platform Namespace Resolution**
- **Problem**: `.cljc` files can't reference `.cljs` namespaces at compile time
- **Symptom**: `Unable to resolve var: handle-event in this context`
- **Solution**: Keep cross-platform tools generic, runtime-specific functions separate
- **Future Tool**: Environment-aware namespace validator

### 2. **ClojureScript vs Clojure API Differences**  
- **Problem**: `requiring-resolve` doesn't exist in ClojureScript
- **Symptom**: `:undeclared-var` warnings in REPL
- **Solution**: Reader conditionals or try/catch with manual requires
- **Future Tool**: CLJS/CLJ API compatibility checker

### 3. **Shadow-cljs Process Conflicts**
- **Problem**: Multiple shadow-cljs processes competing for ports/compilation
- **Symptom**: Mysterious compilation failures, cache corruption
- **Solution**: Automated detection via `check_processes.sh`
- **Status**: ✅ **SOLVED** - Now have prevention tooling

### 4. **Schema Function Discovery Complexity**
- **Problem**: Hard to find which functions have Malli schemas attached
- **Symptom**: Manual `(meta #'function)` calls for each function
- **Solution**: Built `list-known-schemas` and `quick-schema-check` 
- **Future Tool**: Auto-discovery via namespace scanning

### 5. **Development Environment State Confusion**
- **Problem**: Browser/node/REPL state invisible to agents
- **Symptom**: Tools fail silently in wrong environment
- **Solution**: `detect-environment` with clear error messages
- **Status**: ✅ **SOLVED** - Environment detection working

## ClojureScript REPL Gotchas for Future Agents

### Connection Requirements
- **CRITICAL**: Browser must be open at http://localhost:8080 FIRST
- Agent tools load only after successful JS runtime connection
- `(js/console.log "test")` is the definitive connection test

### Data Access Patterns
- ❌ **JS-style**: `store.state.view.selected` (fails silently)
- ✅ **CLJS-style**: `(-> @store :view :selected)`
- Future tool: Pattern detection for wrong data access

### Schema Validation
- Manual validation preferred over automatic instrumentation
- `(meta #'function-var)` for schema inspection
- `(agent/validate-call #'fn args...)` for testing

### File Organization Reality
- `.cljc` files for shared utilities only
- `.cljs` files for browser-specific functionality
- Never reference .cljs from .cljc (compile-time failure)
