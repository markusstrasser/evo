### **8. clj-kondo False Positives with Modern ClojureScript** ⚠️ HIGH
- **Problem**: clj-kondo doesn't understand JS interop contexts or test.check macros
- **Examples**: 
  - `(set! (.-agent js/window.evo) agent)` → "Unresolved symbol: agent"
  - `(defspec test-name ...)` → "Unresolved symbol: test-name" 
- **Root Cause**: Static analysis limitations with dynamic/macro contexts
- **Solution**: Strategic exclusion patterns for known false positives
- **Future Prevention**: Maintain exclusion list for common ClojureScript patterns

### **9. Shadow-cljs CLI Flag Incompatibilities** ⚠️ MEDIUM
- **Problem**: `shadow-cljs compile test --quiet` → "Unknown option: --quiet"
- **Root Cause**: CLI flags vary between shadow-cljs versions/contexts
- **Solution**: Use output redirection instead: `> /dev/null 2>&1`
- **Future Prevention**: Defensive scripting with error handling

### **10. Namespace vs Filename Convention Confusion** ⚠️ MEDIUM  
- **Problem**: clj-kondo expects `reference_tools.cljc` for `agent.reference-tools` namespace
- **Root Cause**: ClojureScript uses kebab-case files, clj-kondo expects snake_case
- **Solution**: Disable `namespace-name-mismatch` linter for ClojureScript projects
- **Impact**: Prevents unnecessary file renaming gymnastics
