# Development Session - Navigation Implementation
*Date: 2025-09-26-10:45*

## Key Failure Modes & Lessons

### **File Naming Issues**
- **Problem**: Namespace with dashes (`navigation-integration-test`) causes FileNotFoundException
- **Solution**: ClojureScript requires files use underscores when namespace has dashes
- **Learning**: Always check file naming conventions before creating test files

### **Environment Mismatch**
- **Problem**: Browser integration tests fail in `:node-test` target
- **Root Cause**: Node environment lacks browser APIs (`js/document`, `js/window`, DOM)
- **Solution**: Separate unit tests (node) from browser integration tests
- **Learning**: Choose test environment based on what you're testing - pure functions vs UI

### **ClojureScript REPL Context**
- **Problem**: Agent namespaces not available in ClojureScript REPL
- **Root Cause**: Agent tools only available in Clojure context, not browser
- **Workaround**: Use browser DevTools JavaScript console for live testing
- **Learning**: Know which REPL context has access to what namespaces

### **Compilation & Hot Reload**
- **Problem**: Code changes not reflected in browser after compilation
- **Root Cause**: Cached build artifacts preventing recompilation
- **Solution**: Force clean recompilation with `rm -rf .shadow-cljs/builds/frontend`
- **Learning**: When changes don't appear, clear build cache and recompile

### **ClojureScript Data Structure Access**
- **Problem**: Accessing ClojureScript maps from JavaScript console is non-trivial
- **Root Cause**: ClojureScript data structures have complex internal representation
- **Solution**: Use `cljs.core.clj__GT_js()` conversion or direct property access
- **Learning**: Browser debugging requires understanding ClojureScript/JS interop

### **Store State Inspection**
- **Problem**: Difficulty accessing nested store state from browser console
- **Root Cause**: Atom dereferencing and nested map access requires ClojureScript functions
- **Solution**: Use proper ClojureScript accessors or conversion functions
- **Learning**: Plan debugging strategy for ClojureScript data structures in browser

## Effective Debugging Workflow

1. **Unit Tests**: Fast feedback for pure functions (node environment)
2. **Browser Console**: Live testing of compiled ClojureScript (proper environment)
3. **Force Recompilation**: Clear cache when changes don't appear
4. **ClojureScript Inspection**: Use conversion functions for complex data structures
5. **Environment Matching**: Test in the environment where code will run

## Key Takeaway
**Test the data layer in appropriate environment, verify integration in browser**
