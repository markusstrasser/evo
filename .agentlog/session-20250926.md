# Agent Session Log - September 26, 2025

## Failure Modes Encountered

### 1. **Invisible State Structure Mismatches**
**Problem**: Schema validation errors were cryptic - "missing required key :selected" didn't clearly indicate the new selection structure needed `:selection`, `:selection-set`, `:cursor`.

**Fix Needed**: 
- Schema validation should show **expected vs actual structure**
- Tool to validate state shape changes across all consumers (renderer, tests, etc.)

### 2. **Event Handling Context Loss in Tests**
**Problem**: `select-node` command worked in UI but failed in tests because tests passed `{}` instead of proper DOM event, causing `(modifiers event)` to return `nil` and exit early.

**Fix Needed**:
- Mock event utilities for testing UI commands
- Commands should gracefully handle missing event context
- Test helper that provides realistic event objects

### 3. **Infinite Watch Loop Detection**
**Problem**: Store watchers created infinite loops by calling `reset!` inside watch functions. No clear indication this was happening until manual debugging.

**Fix Needed**:
- Watch loop detection tool (warn after N rapid updates)
- Store debugging utilities that trace update chains

### 4. **Test/UI Dispatch Path Divergence**
**Problem**: Tests used `dispatch-command` while UI used `dispatch-commands`. Navigation commands failed because tests didn't set `:cursor` field that UI navigation depends on.

**Fix Needed**:
- Unified test utilities that mirror exact UI behavior
- Store state validators that ensure required fields are set
- Test helpers that automatically set all three selection fields consistently

### 5. **ClojureScript REPL Context Confusion**
**Problem**: Tried to debug in REPL but browser wasn't connected. ClojureScript evaluation failed with "FileNotFoundException" because namespaces weren't loaded in browser context.

**Fix Needed**:
- Environment detection before REPL attempts
- Better error messages when browser runtime missing
- Auto-open browser tab when starting REPL session

### 6. **Hidden Dependencies in Navigation Functions**
**Problem**: Navigation commands silently returned `nil` with no error indication. Root cause was missing `:cursor` in test state, but this wasn't obvious.

**Fix Needed**:
- Navigation function validation (warn when cursor is nil)
- Better debug output for command execution chains
- Test state validators that check prerequisites

## Smart Tooling Recommendations

1. **State Shape Validator**: Tool that compares current state against schema and shows diffs
2. **Command Execution Tracer**: Shows full path from dispatch → command → state change
3. **Test Context Mirror**: Utilities that exactly replicate UI environment in tests
4. **Watch Loop Detector**: Automatic detection of infinite update cycles
5. **REPL Environment Checker**: Validates browser connection before namespace loading

## Key Learnings

- **ClojureScript testing requires careful event context setup**
- **Selection state needs **three** synchronized fields for deterministic behavior**
- **Watch functions must never modify the atom they're watching**
- **Schema errors hide structural mismatches - need better diff tools**
- **Navigation commands are fragile without proper cursor state**
