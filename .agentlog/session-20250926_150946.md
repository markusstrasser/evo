# Agent Session - DX Improvements Implementation
## September 26, 2025 15:09

### New Failure Modes Discovered

#### 1. **Malli Validation Compilation Errors**
**Problem**: Using `(validate schema data)` instead of `(m/validate schema data)` in CLJC files causes compilation failures
**Tooling Fix**: Use fully qualified Malli function calls, avoid unqualified imports

#### 2. **JavaScript Literal Syntax in Reader Conditionals**
**Problem**: `#js (merge {...})` fails compilation - can't merge Clojure data directly into JS literals
**Tooling Fix**: Use intermediate let bindings: `(let [data (merge ...)] #js {:key (clj->js data)})`

#### 3. **Missing clojure.set Imports**
**Problem**: Using `set/difference` without requiring `[clojure.set :as set]` causes unresolved namespace errors
**Tooling Fix**: Always explicitly require set operations, don't rely on implicit imports

#### 4. **Shadow-cljs Process Conflicts During Development**
**Problem**: Running `npm dev` + manual `shadow-cljs` commands creates process conflicts, breaks compilation
**Tooling Fix**: Enhanced process detection scripts, automated conflict resolution

#### 5. **Agent Tool Integration Confusion**
**Problem**: Uncertainty about which DX tools already exist vs need to be created
**Tooling Fix**: Better documentation of existing capabilities, clear inventory of agent namespace functions

#### 6. **ClojureScript vs CLJ Context Confusion**
**Problem**: Agent trying to implement CLJ-specific file operations (slurp) in CLJC files targeting browser
**Tooling Fix**: Stricter environment detection, clearer platform capability boundaries

### Development Environment Insights

#### Strong Points
- ✅ Comprehensive test coverage catches integration issues
- ✅ Pre-commit validation prevents syntax errors reaching git
- ✅ Agent utilities already handle most common debugging scenarios
- ✅ Shadow-cljs hot reload works reliably when process conflicts avoided

#### Weak Points  
- ❌ No automated detection of missing requires/imports
- ❌ Process conflict detection only warns, doesn't auto-resolve
- ❌ Agent namespace inventory scattered across multiple files
- ❌ No unified "debug state" command that runs all checks

### Recommendations for Future Agents

#### Project Structure
1. **Consolidate Agent Documentation**: Single `AGENT-TOOLS.md` with complete function inventory
2. **Standardize Import Patterns**: Always use explicit requires, never rely on implicit imports
3. **Environment-Specific Modules**: Split browser vs node functionality clearly in CLJC files

#### Development Workflow
1. **One-Command Health Check**: `npm run agent-check` that runs all DX validations
2. **Auto-Conflict Resolution**: Make process detection script kill conflicts automatically
3. **Required Import Linting**: Clj-kondo rules to catch missing set/data/string requires

#### Naming Conventions
1. **Function Naming**: Prefix all agent tools with verb: `validate-*`, `check-*`, `trace-*`
2. **Namespace Organization**: Keep utilities grouped by purpose (validation, debugging, testing)
3. **Error Messages**: Always include "suggestion" field with actionable fix

### Key Learnings

**Shadow-cljs Process Management**: Multiple shadow-cljs processes are the #1 cause of mysterious compilation failures. Always check for conflicts first.

**CLJC Platform Boundaries**: Reader conditionals hide complexity. Be explicit about what works where.

**Agent Tool Discovery**: Without clear inventory, agents waste time reimplementing existing functionality.

**ClojureScript Compilation Strictness**: Much stricter than CLJ about syntax, imports, and JS interop patterns.
