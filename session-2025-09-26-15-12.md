## Trouble Encountered & Lessons Learned

### 1. **Namespace/Path Mismatch**
**Failure Mode**: shadow-cljs config had `"agent"` as source path but files were in `src/agent/`
**Symptom**: "namespace not available" errors during compilation
**Fix**: Remove incorrect source path from shadow-cljs.edn

### 2. **Missing Dependencies in .cljc Files**
**Failure Mode**: store-inspector.cljc used `clojure.set` and `clojure.string` without requiring them
**Symptom**: Compilation fails with "namespace not available"
**Fix**: Add explicit `:require` declarations

### 3. **Environment Detection Logic Error**
**Failure Mode**: `detect-environment` looked for `evolver.state/store` instead of `evolver.core/store`
**Symptom**: `store-accessible?` always returned false even when store was available
**Fix**: Update `resolve` call to correct namespace

### 4. **Build Target Inclusion**
**Failure Mode**: Agent namespaces not compiled into `:frontend` build target
**Symptom**: Files exist but can't be required in browser REPL
**Fix**: Add agent namespaces to main evolver.core requires

### 5. **REPL Context vs Build Context**
**Failure Mode**: Namespaces work in `:test` target but not `:frontend` target
**Symptom**: `check test` passes but `compile frontend` fails
**Learning**: Different build targets have different classpath visibility
