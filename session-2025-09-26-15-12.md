## Trouble Encountered & Lessons Learned

### 1. **Namespace/Filename Mismatch Hell**
**Failure Mode**: Shadow-cljs requires exact filename matches - `some_file.cljc` must contain `(ns some_file)`, not `(ns some-file)`
**Solution**: Either use all underscores or all dashes consistently
**Key Learning**: ClojureScript tooling is strict about filename conventions, unlike Clojure JVM

### 2. **Source Path vs Build Target Confusion** 
**Failure Mode**: Files in global `:source-paths` don't automatically compile into build targets
**Solution**: Move agent tools to `src/agent/` instead of standalone `./agent/` directory
**Key Learning**: shadow-cljs only compiles what's explicitly required or in standard source paths

### 3. **Browser Runtime Connection Required**
**Failure Mode**: "No available JS runtime" when trying ClojureScript REPL without browser
**Solution**: Must open `http://localhost:8080` first, then connect REPL
**Key Learning**: ClojureScript REPL needs active browser tab for evaluation context

### 4. **Cache Corruption Masking Real Issues**
**Failure Mode**: Mysterious "namespace not found" errors persisting after fixes
**Solution**: `npx shadow-cljs stop && rm -rf .shadow-cljs out target` 
**Key Learning**: Always clean cache when debugging namespace resolution

### 5. **Node Test vs Browser Context Gap**
**Failure Mode**: Agent tests fail in `:node-test` target but work in browser
**Solution**: Separate pure logic tests (node) from browser integration tests (Chrome DevTools)
**Key Learning**: Testing strategy must match execution environment

## Dev Tooling Architecture Insights

### What Works Well:
- Chrome DevTools MCP integration for live browser testing
- JavaScript injection for runtime debugging (`window.evo.*` functions)
- ClojureScript REPL for interactive development once connected

### What's Fragile:
- Namespace resolution during compilation
- Cache state between builds  
- Browser/REPL connection lifecycle
- File naming conventions

### Critical Dependencies:
- Browser tab must stay open for REPL
- shadow-cljs server must be running
- Clean cache needed when changing file structure
- Exact filename/namespace matching required
