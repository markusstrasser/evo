How to build a tournament-evolution system for code that avoids convergence to judge consensus mediocrity when judges lack calibration data?

Three scenarios where this breaks:

Sophistication creep: Each generation gets more "clever" but harder to maintain (judges reward novelty over simplicity)
Mode collapse: System converges to one style that judges love but violates goals.md in subtle ways
Circular reasoning: Generator learns to game judges, judges calibrate to generator output
