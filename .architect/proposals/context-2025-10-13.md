# EVO Review Process: Ideas → Proposals → Specs → ADRs

## Current Workflow

### 1. Idea Generation
- Manual brainstorming
- Ad-hoc notes in `.archived/ideas/`
- No structured format

### 2. Architectural Proposals (Automated)
**Script**: `bb architectural-proposals`

**Process**:
1. **Generate 15 proposals** (5 questions × 3 AI providers: Gemini, Codex, Grok)
2. Uses **project overview** (not source code) to avoid implementation bias
3. **Ranks proposals** (Codex + Grok) against project criteria

**Questions Target**:
- Core kernel operations
- Derived indexes
- Transaction pipeline
- Extensibility/composition
- Developer experience

**Output Structure**:
```
research/proposals/YYYY-MM-DD-HH-MM/
├── proposals/
│   ├── {provider}-{question}.md  (15 files)
└── rankings/
    ├── codex-ranking.md
    └── grok-ranking.md
```

**Ranking Criteria**:
1. **Simplicity** - Reduces conceptual complexity
2. **Readability** - Human/LLM can understand quickly
3. **Debuggability** - Observable, clear errors, REPL-friendly
4. **Expressiveness** - Enables powerful composition, low LoC
5. Novel concepts (if they unlock expressiveness)
6. DX (REPL-friendly, AI assistant-friendly)

**Ignores**: Performance, feature completeness, backwards compatibility

### 3. Refactor Analysis (Battle-Test)
**Script**: `bb best-repos-research`

**Process**:
1. Query 5 reference repos (datascript, re-frame, meander, specter, reitit)
2. Use 3 AI providers per repo (15 parallel queries)
3. Ask: "How does this repo implement X?"
4. Synthesize patterns with Codex reasoning

**Output**:
```
research/results/YYYY-MM-DD-HH-MM/
├── {repo}-{model}.md  (15 files)
└── analysis.md        (Codex synthesis)
```

### 4. Specs/ADRs (Manual)
- No structured process yet
- Proposals/analysis → manual code changes
- No formal ADR documentation

## Current Pain Points

**Disconnected Steps**:
- Proposals → Implementation is manual
- No tracking of: which proposals were tried, which worked, which failed
- No connection between proposals and actual code changes
- Rankings are advisory only, not actionable

**No Decision Records**:
- Why we chose one approach over another?
- What tradeoffs were considered?
- What experiments failed?

**No Spec Stage**:
- Proposals are high-level (good for avoiding impl bias)
- But jump straight from proposal → code
- Missing: concrete spec with signatures, examples, test cases

**Tool Proliferation**:
- `bb architectural-proposals` - proposal generation
- `bb best-repos-research` - pattern mining
- Manual synthesis
- Manual implementation

## Design Constraints

**Project Philosophy** (from CLAUDE.md):
- Solo developer + AI agents
- Focus on 80/20, not performance
- Simplicity over optimization
- Kernel transaction architecture is core (create, place, update)
- REPL-driven development
- Data-oriented (immutability, explicit data flow)

**Quality Gates**:
- Pre-commit: linting, tests, namespace validation
- Core modules separate from shell concerns
- Instrumentation focus for observability

## Ideal Flow (Not Yet Implemented)

```
Ideas → Proposals → Ranking → Specs → Implementation → ADRs
  ↓         ↓          ↓         ↓           ↓            ↓
 Notes   15 MDs    Scored   Concrete    Code PR      Decision
          ↓                  Types                    Records
       Research              Tests
      (5 repos)             Examples
```

## Questions for AI

2. **How to leverage MCP patterns?**
   - Resources for proposals/rankings?
   - Tools for generation/analysis?

# FastMCP 2.x API Reference (Terse)

## Core Primitives

### Server
```python
from fastmcp import FastMCP, Context

mcp = FastMCP("ServerName")

# Composition
mcp.mount("/prefix", other_mcp_server)  # Live link
mcp.import_server(other_mcp_server)     # Static copy

# Proxy
mcp = FastMCP.as_proxy("http://remote/mcp")
```

### Tools (POST-like, side effects)
```python
@mcp.tool
async def process(param: str, ctx: Context) -> dict:
    """Docstring becomes description"""
    await ctx.info("Progress update")
    return {"result": "value"}
```

### Resources (GET-like, read-only)
```python
# Static
@mcp.resource("config://version")
def version(): return "1.0"

# Dynamic template with {placeholders}
@mcp.resource("users://{user_id}/profile")
def profile(user_id: int): return {...}

# RFC 6570 query params (NEW in 2.14)
@mcp.resource("search://{?q,limit}")
def search(q: str, limit: int = 10): return [...]
```

### Prompts (interaction templates)
```python
@mcp.prompt
def summarize(text: str) -> str:
    return f"Summarize: {text}"

# With Message objects
from fastmcp.prompts import PromptMessage

@mcp.prompt
def structured(data: dict) -> list[PromptMessage]:
    return [PromptMessage(role="user", content=...)]
```

### Context (injected automatically)
```python
@mcp.tool
async def tool(param: str, ctx: Context):
    # Logging
    await ctx.info("message")
    await ctx.error("error")

    # LLM sampling (ask client's LLM)
    response = await ctx.sample("prompt")

    # Resource access
    data = await ctx.read_resource("uri")

    # Progress reporting
    await ctx.report_progress(50, 100)

    # HTTP requests
    resp = await ctx.http_request("GET", "https://api.example.com")
```

## Client

### Basic Usage
```python
from fastmcp import Client

# Auto-detect transport
async with Client("server.py") as client:  # stdio
async with Client("http://host/mcp") as client:  # HTTP
async with Client(mcp_instance) as client:  # in-memory

# Use client
tools = await client.list_tools()
result = await client.call_tool("name", {"arg": "value"})
resources = await client.list_resources()
content = await client.read_resource("uri")
```

### Multi-Server Config
```python
config = {
    "mcpServers": {
        "server1": {"command": "python", "args": ["s1.py"]},
        "server2": {"url": "https://remote/mcp"}
    }
}

async with Client(config) as client:
    # Tools/resources prefixed with server name
    await client.call_tool("server1_tool_name", {...})
```

## Advanced Patterns

### Server Composition
```python
# Modular server design
auth_mcp = FastMCP("auth")
@auth_mcp.tool
def login(...): ...

data_mcp = FastMCP("data")
@data_mcp.tool
def query(...): ...

# Combine
main_mcp = FastMCP("main")
main_mcp.mount("/auth", auth_mcp)
main_mcp.mount("/data", data_mcp)
```

### Proxy Server
```python
# Add logic layer to existing server
proxy = FastMCP.as_proxy("http://upstream/mcp")

@proxy.tool
def enhanced_tool(...):
    # Add caching, logging, validation, etc.
    ...
```

### OpenAPI/FastAPI Integration
```python
# From OpenAPI spec
mcp = FastMCP.from_openapi("https://api.example.com/openapi.json")

# From FastAPI app
from fastapi import FastAPI
app = FastAPI()
mcp = FastMCP.from_fastapi(app)
```

## Authentication (Enterprise OAuth)

### Server Side
```python
from fastmcp.server.auth import GoogleProvider, GitHubProvider

auth = GoogleProvider(
    client_id="...",
    client_secret="...",
    base_url="https://myserver.com"
)

mcp = FastMCP("Protected", auth=auth)
```

### Client Side
```python
# Auto OAuth flow (browser-based)
async with Client("https://protected/mcp", auth="oauth") as client:
    ...
```

**Providers**: Google, GitHub, Azure, Auth0, WorkOS, Descope, Scalekit, AWS Cognito, JWT, API Keys


### HTTP/SSE
```python
mcp.run(transport="http", host="0.0.0.0", port=8000)
mcp.run(transport="sse", host="0.0.0.0", port=8000)
```

### CLI
```bash

```

## Recent Features (2.14+)

- **RFC 6570 query parameters** in resource templates: `resource://path{?param1,param2}`
- **Scalekit/AWS Cognito** auth providers
- **Python 3.13** support
- **OAuth 2.1** spec compliance
- **FastMCP.json schema** for configuration
- **Context arguments** in client completions

## Key Patterns

**Minimal Tool**:
```python
from fastmcp import FastMCP
mcp = FastMCP("Demo")

@mcp.tool
def add(a: int, b: int) -> int:
    return a + b

mcp.run()
```

**Contextual Tool**:
```python
@mcp.tool
async def process(uri: str, ctx: Context):
    data = await ctx.read_resource(uri)
    summary = await ctx.sample(f"Summarize: {data}")
    return summary.text
```

**Resource Template**:
```python
@mcp.resource("db://{table}/rows{?limit,offset}")
def query(table: str, limit: int = 10, offset: int = 0):
    return fetch_rows(table, limit, offset)
```

**Composition**:
```python
main = FastMCP("main")
main.mount("/tools", tools_mcp)
main.mount("/data", data_mcp)
main.run()
```
